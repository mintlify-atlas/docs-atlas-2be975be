`BrokerMessage<T>` is the envelope type that Broccoli uses to wrap user-defined payloads. Every message published to or consumed from a queue is represented as a `BrokerMessage<T>`, giving you access to the task identifier, attempt counter, and fairness disambiguator alongside your payload.

## BrokerMessage\<T\>

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct BrokerMessage<T: Clone> {
    pub task_id: uuid::Uuid,
    pub payload: T,
    pub attempts: u8,
    pub disambiguator: Option<String>,
}
```

### Fields

<ResponseField name="task_id" type="uuid::Uuid">
  A UUID v4 assigned automatically when the message is first created via `BrokerMessage::new`. Remains stable across retries — the same task_id is preserved on each redelivery.
</ResponseField>

<ResponseField name="payload" type="T">
  The user-defined message content. `T` must implement `Clone + Serialize + DeserializeOwned`.
</ResponseField>

<ResponseField name="attempts" type="u8">
  The number of times this message has been delivered. Starts at `0` on the first delivery and increments by 1 on each retry.
</ResponseField>

<ResponseField name="disambiguator" type="Option<String>">
  An optional string identifier used for fairness queue routing. Set when publishing with a non-`None` disambiguator. `None` for messages published to standard queues.
</ResponseField>

### Using BrokerMessage in Handlers

```rust
use broccoli_queue::brokers::broker::BrokerMessage;

queue.process_messages(
    "jobs",
    Some(4),
    None,
    |msg: BrokerMessage<JobPayload>| async move {
        println!("Task ID:      {}", msg.task_id);
        println!("Attempt:      {}", msg.attempts);
        println!("Payload:      {:?}", msg.payload);
        println!("Group:        {:?}", msg.disambiguator);

        if msg.attempts >= 2 {
            // Final attempt — escalate
            send_alert(&msg.payload).await;
        }

        process_job(msg.payload).await
    },
).await?;
```

### Constructor

```rust
pub fn new(payload: T, disambiguator: Option<String>) -> Self
```

Creates a new `BrokerMessage` with a fresh UUID v4 `task_id` and `attempts` set to `0`. Normally called internally by Broccoli when you call `publish()` — you don't need to create `BrokerMessage` instances manually.

## QueueStatus *(management feature)*

`QueueStatus` is returned by `BroccoliQueue::queue_status()` and provides a snapshot of a queue's current state. Available only with `features = ["management"]`.

```rust
#[derive(Debug, Clone)]
pub struct QueueStatus {
    pub name: String,
    pub queue_type: QueueType,
    pub size: usize,
    pub processing: usize,
    pub failed: usize,
    pub disambiguator_count: Option<usize>,
}
```

### Fields

<ResponseField name="name" type="String">
  The name of the queue.
</ResponseField>

<ResponseField name="queue_type" type="QueueType">
  The type of the queue: `Main`, `Processing`, `Failed`, or `Fairness`.
</ResponseField>

<ResponseField name="size" type="usize">
  Number of messages currently waiting in the main queue.
</ResponseField>

<ResponseField name="processing" type="usize">
  Number of messages currently being processed (in the processing queue).
</ResponseField>

<ResponseField name="failed" type="usize">
  Number of messages in the failed queue (exhausted all retries).
</ResponseField>

<ResponseField name="disambiguator_count" type="Option<usize>">
  For fairness queues, the number of distinct disambiguator sub-queues. `None` for standard queues.
</ResponseField>

## QueueType Enum

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum QueueType {
    Failed,      // display: "failed"
    Processing,  // display: "processing"
    Main,        // display: "main"
    Fairness,    // display: "fairness"
}
```

All variants implement `Display` via `derive_more::Display`. Use `format!("{}", queue_type)` to get the string representation.
