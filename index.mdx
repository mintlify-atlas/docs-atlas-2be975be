Broccoli gives Rust applications a simple, type-safe API for publishing and consuming messages across distributed systems. Built on Tokio, it supports multiple message brokers, configurable retry strategies, priority queuing, message scheduling, and fairness-aware consumption — all with minimal boilerplate.

## Why Broccoli?

If you've used Celery in Python and want the same ergonomics with Rust's performance and compile-time safety, Broccoli is for you. It abstracts broker-specific details behind a single unified interface, so switching from Redis to RabbitMQ requires only a URL change.

## Get Started

<Steps>
  <Step title="Add Broccoli to Cargo.toml">
    ```toml Cargo.toml
    [dependencies]
    broccoli_queue = "0.4"
    tokio = { version = "1", features = ["full"] }
    serde = { version = "1", features = ["derive"] }
    ```
  </Step>
  <Step title="Create a queue instance">
    ```rust
    use broccoli_queue::queue::BroccoliQueue;

    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .pool_connections(5)
        .failed_message_retry_strategy(Default::default())
        .build()
        .await?;
    ```
  </Step>
  <Step title="Publish a message">
    ```rust
    queue.publish("jobs", None, &my_payload, None).await?;
    ```
  </Step>
  <Step title="Consume and process messages">
    ```rust
    queue.process_messages("jobs", Some(4), None, |msg| async move {
        println!("Got: {:?}", msg.payload);
        Ok(())
    }).await?;
    ```
  </Step>
</Steps>
